# 表单名称和表单编码字段迁移至表单设计标签页

> 编码: 958191 | 日期: 2026-04-28 | 调整类型: 重构 | 状态: 草稿

## 调整背景

流程设计界面现在的表单名称和表单编码是在基本信息界面进行编辑的，我希望可以调整到表单设计界面。在表单设计界面顶部添加一行将这两个字段放在这里呈现。

## 关键要点

- **涉及模块**:
  - `@coding-flow/flow-design`（流程设计器组件库，核心变更模块）
    - `components/design-panel/tabs/base/` — 基本信息标签页，需移除表单名称和表单编码字段
    - `components/design-panel/tabs/form/` — 表单配置标签页，需在顶部新增一行展示这两个字段
    - `components/design-panel/presenters/` — MVP Presenter 层，可能涉及状态管理调整
- **影响范围**:
  - DesignPanel 组件的标签页布局结构变更
  - 基本信息标签页的字段集合减少（移除 title/code 相关输入项）
  - 表单设计标签页的顶部布局增加新行
  - Presenter 层状态同步逻辑可能需调整（字段从 base tab 迁移到 form tab 的状态绑定）
- **前置条件/迁移建议**:
  - 需确认表单名称和表单编码字段在 Presenter/Redux 中的数据路径，迁移时保持数据绑定不变
  - 需确认基本信息标签页移除这两个字段后是否还有其他字段，若无则需考虑标签页是否保留
  - UI 布局建议：表单设计标签页顶部新增一行，水平排列表单名称和表单编码两个输入框
